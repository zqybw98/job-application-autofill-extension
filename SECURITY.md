# Security Policy

## Data Safety Rules

- Do not commit personal profile data.
- Do not commit CVs, cover letters, certificates, or application records.
- Do not commit `.env`, `.env.local`, `profile.json`, secrets, credentials, or private tokens.
- Use only fake data in examples.

## Extension Safety Rules

- Keep the extension privacy-first.
- Do not add telemetry or analytics.
- Do not add external API calls without an explicit user request.
- Do not bypass login, CAPTCHA, upload, or application platform restrictions.
- Do not implement automatic application submission.

## Reporting Security Issues

If this project is published, report security issues through the repository maintainer or a private GitHub security advisory. Do not disclose real personal data in reports.
